# notes/ — research garden

Evolving, **interlinked** research notes in the "digital garden" style: organized by topic
and association rather than by date, and deliberately allowed to be half-finished and to
**grow over time**. This is where build-in-public thinking lives *before* it is distilled
into the canonical [`../spec/`](../spec/).

## Conventions

- **One concept per file**, named in kebab-case (`waterfall-ordering.md`, `tranche-token-layer.md`).
- **Link liberally** between notes with relative Markdown links — a link to a note that
  doesn't exist yet is fine; it marks something worth writing later.
- **Cite sources inline.** Prefer real, fetched primary sources over recollection.
- Keep an [`index.md`](./index.md) as the map of the garden.
- When a note matures into normative spec material, move the conclusion into `spec/` and
  leave a back-link here.

See [`index.md`](./index.md) for the current map.
